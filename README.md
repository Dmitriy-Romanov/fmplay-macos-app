# FMPLAY macOS

Версия: `1.3.0`

FMPLAY macOS - небольшое нативное приложение для macOS, написанное на Rust, для прослушивания радиостанций из каталога [`fmplay.ru`](https://fmplay.ru/).

Приложение загружает актуальный список станций с FMPLAY, позволяет быстро искать радио по названию, переключать качество потока `16 / 24 / 32 / 48` кбит/с, выбирать тему оформления, добавлять станции в избранное, видеть текущую композицию на панели и в заголовке окна.

Интерфейс спроектирован для повседневного прослушивания и имеет премиальный дизайн: слева находится компактная панель плеера с обложкой, эргономичной объединенной кнопкой Play/Stop, избранным, настройками и громкостью, а справа — список станций с поиском. Окно можно свернуть до узкого режима (только плеер слева) и развернуть обратно (показать каталог станций справа) с помощью удобной кнопки-стрелки (`◀` / `▶`).

## Возможности

- Каталог радиостанций FMPLAY загружается автоматически.
- Избранные станции поднимаются вверх списка и сохраняются между запусками.
- Опция **"Только избранное"** в настройках позволяет оставить в списке исключительно любимые станции.
- Последняя выбранная станция, качество потока, громкость, тема, размер окна и фильтр избранного сохраняются локально.
- Отображение текущего трека на панели воспроизведения и динамически в заголовке окна macOS.
- Объединенная кнопка Play/Stop для интуитивного управления.
- **Премиальная визуализация и анимации**:
  - Анимированный динамический CSS-эквалайзер для играющей станции прямо в списке.
  - Мягкое органическое "дыхание" (пульсация масштаба до 1.06x) неонового свечения вокруг обложки текущей станции.
  - Плавные анимации интерфейса на основе нативных физических кривых (`cubic-bezier` в стиле Apple Spring).
- Удобное и быстрое переключение тем оформления и опций в окне настроек.
- Легковесность: готовое приложение компилируется в один бинарник размером **менее 1 МБ** (около 900 КБ).
- Можно собрать полноценный `.app` bundle с иконкой FMPLAY.

## Запуск

```sh
cargo run
```

Настройки приложения сохраняются в:

```text
~/Library/Application Support/FMPLAY Radio/config.json
```

## Сборка

```sh
cargo build --release
```

Готовый бинарник будет в `target/release/fmplay_macos_app`.

## macOS app

Чтобы собрать `.app` с иконкой FMPLAY:

```sh
scripts/build_app.sh
```

Готовое приложение будет в:

```text
target/release/FMPLAY Radio.app
```

### Если macOS блокирует приложение

После скачивания release с GitHub macOS может оставить на приложении quarantine attribute. Если `.app` не запускается на другом Mac, выполните:

```sh
xattr -rd com.apple.quarantine "FMPLAY Radio.app"
```

## GitHub Actions и релизы

В репозитории настроен GitHub Actions workflow:

- при push в `master` собирается macOS `.app` и загружается build artifact;
- при push тега вида `v1.3.0` дополнительно создается GitHub Release;
- в release прикладывается архив `FMPLAY Radio-macOS.zip`.

Чтобы выпустить релиз:

```sh
git tag v1.3.0
git push origin v1.3.0
```
